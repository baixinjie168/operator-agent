核心判断发生在 OptionalExtract 节点：LLM 根据参数的 llm_description 文本和一组关键词清单，返回 true 或 false。

1 FunctionSignatureExtract LLM

packages/agent/src/agent/nodes/function_signature_extract.py

输入：文档的 function_prototype 章节（C++ 函数原型代码块）。

处理：LLM 使用 FUNCTION_SIGNATURE_EXTRACT_PROMPT 提取结构化签名，包括函数名、返回类型、参数列表。

输出：扁平化的参数列表，每条记录包含：

{
  "function_name": "aclnnConvolutionBackwardGetWorkspaceSize",
  "param_name": "weight",
  "param_type": "aclTensor"
}

与 is_optional 的关系：此步骤不涉及 is_optional，仅产出参数名和类型，供下游节点使用。

packages/agent/src/agent/nodes/table_column_extract.py

输入：文档的 params_get_workspace / params_execute 章节中的 HTML 参数表格。

处理：零 LLM 调用，直接从 8 列表格中解析各列。

提取字段：

表格列映射字段

与 is_optional 的关系：此步骤不提取 is_optional。参数表格中没有专门的“是否可选”列。

3 LlmDescriptionExtract LLM

packages/agent/src/agent/nodes/llm_description_extract/

这是一个子图（sub-graph），内部流程为：

fetch_sections
↓
extract_ws
extract_exe
↓
validate_results → verify_and_enhance → save_descriptions

3.1 fetch_sections

从 DB 获取以下章节内容，按 GetWorkspaceSize / Execute 分组：

  • params_get_workspace — 参数表格
  • return_codes_get_workspace — 返回值/错误码
  • constraints — 约束说明

3.2 extract_ws / extract_exe

对每个参数：

  1. 调用 extract_param_context() 从完整章节文本中裁剪出与该参数相关的上下文（500~2000 字符）
  2. 将上下文填入 LLM_DESCRIPTION_EXTRACT_PROMPT，发送给 LLM
  3. LLM 返回 JSON：
{
  "llm_description": "weight是卷积核权重张量，对应公式中的w。支持空Tensor。...",
  "src_content": "（原始文档文本）",
  "direction": "输入",
  "is_support_discontinuous": true
}

3.3 save_descriptions

将结果写入 DB 的 parameters 表，并将 llm_description 合并回 state.parameters，供下游节点读取。

ⓘ 与 is_optional 的关系：
此步骤不直接设置 is_optional，但其产出的 llm_description 文本是 OptionalExtract 的唯一输入源。
描述中是否包含关键词，直接决定了 is_optional 的取值。

数据流：
章节文本 → extract_param_context() 裁剪 → LLM_DESCRIPTION_EXTRACT_PROMPT → LLM → llm_description 传给 Step 4

4 OptionalExtract LLM

packages/agent/src/agent/nodes/optional_extract.py

这是 is_optional 取值的决定性步骤。

4.1 过滤

从 state.parameters 中筛选出 llm_description 非空的参数：

# optional_extract.py:53
described = [p for p in params if p.get("llm_description")]

4.2 逐参数 LLM 调用

对每个参数，调用 _extract_optional()：

  1. 检查 param_name 是否包含 "optional"（不区分大小写）→ 传入 name_has_optional 标志 
    # optional_extract.py:98
name_has_optional = "optional" in param_name.lower()
  2. 将以下变量填入 OPTIONAL_EXTRACT_PROMPT：
    • {param_name} — 参数名
    • {name_has_optional} — "true" / "false"
    • {params_text} — 来自 Step 3 的 llm_description
  3. LLM 返回 JSON：{"param_name": "weight", "is_optional": true}

4.3 值转换

LLM 返回的布尔值被转换为整数：

# optional_extract.py:112
result["is_optional"] = 1 if val is True or val == "true" else 0

4.4 写入 DB

通过 MCP 调用 update_param_optional，将 is_optional（0 或 1）写入 parameters 表：

-- DB schema (db.py:38)
ALTER TABLE parameters ADD COLUMN is_optional INTEGER NOT NULL DEFAULT 0
⚠ 关键点：
is_optional 的判断完全依赖 LLM 对 llm_description 文本的理解，没有规则兜底。
LLM 的关键词匹配结果直接成为最终值，且 src_text 始终为空 — 无法溯源。

OPTIONAL_EXTRACT_PROMPT 完整内容

packages/agent/src/agent/prompts/__init__.py:178-198

你是一个参数可选性判断专家。从下面参数的描述文本中，判断该参数是否可选。

关键词清单（描述中出现以下任一表达即视为可选）:
- 中文: 可选、可空、可为空、可不传、不传、非必选、缺省值、默认值、默认
- 英文: Optional、optional、default、None、null、nullable
- 其他: 可以为 nullptr、可为 nullptr、空指针、未指定时

重要规则:
- 参数名为 "{param_name}"
- 参数名包含 Optional: {name_has_optional}
- 如果参数名包含 Optional，请忽略参数名，只看描述内容
- 如果描述内容中确实出现了上述关键词，返回 true，否则返回 false
- 请注意：如果描述中出现"必选"、"必须"、"不可为空"等字眼，则不是可选，返回 false

严格按以下JSON格式返回，不要添加任何其他文字:
{"param_name": "{param_name}", "is_optional": true}

参数描述:
{params_text}

其中 {params_text} 即为 Step 3 产出的 llm_description。

已知误判问题：“空指针”与“支持空Tensor”

❌ 问题：关键词 "空指针" 导致 false positive

Prompt 关键词清单中包含 “空指针”，意图是匹配 “可以为 nullptr” 这类表示参数可选的表达。但在实际文档中，“空指针”更常见于返回值错误码章节，描述的是“传空指针会报错” — 这恰恰说明参数是必选的。

典型误判案例：aclnnConvolutionBackward 的 weight 参数

文档原文 语义 LLM 误判
错误码表：传入的gradOutput、input、weight、...是空指针 → 报错 ACLNN_ERR_PARAM_NULLPTR weight 传空指针会报错 → 必选 看到 "空指针" → is_optional: true ✗
参数表使用说明：支持空Tensor 允许 shape 维度为 0 的 Tensor → 必选（指针不能为 null） 可能混淆 "空Tensor" 与 "可选" → is_optional: true ✗

✅ 正确判断的依据（原文印证）

以下是 aclnnConvolutionBackward 文档中可以印证 weight is_optional = false 的原文：

# 原文位置 原文内容 说明
1 错误码表 传入的gradOutput、input、weight、...是空指针 → 报错 161001 传空指针会报错，说明必选
2 参数表 weight | 输入 | 公式中的w。 无任何“可选”/“Optional”标注
3 参数表使用说明 支持空Tensor。 允许 shape 含 0 维度的 Tensor，但指针本身必须有效
4 函数原型 const aclTensor *weight, 普通指针参数，无 optional 注释或默认值
5 约束说明 H、W的大小应该在[1,255]的范围内，其他维度的大小应该大于等于1 对 weight 各维度有明确约束，说明必须有值

⚠ 根因总结

  1. 关键词语义歧义：“空指针”在关键词清单中意为“可以为 nullptr → 可选”，但在文档中更常出现在“传空指针报错 → 必选”的语境中。
  2. “支持空Tensor” ≠ “参数可选”：空Tensor 是 shape 维度为 0 的合法 Tensor 对象，参数指针本身仍必须传入。
  3. 无规则兜底：is_optional 完全依赖 LLM 判断，没有基于错误码表或函数原型的规则校验。
  4. src_text 缺失：OptionalExtract 未要求 LLM 返回判断依据的原文，导致无法溯源和自动纠错。

💡 改进建议

  1. 从关键词清单中移除 “空指针”，或将规则改为：
    • “传空指针报错” / “是空指针时报错” → is_optional = false
    • “可以为 nullptr” / “支持 nullptr” → is_optional = true
  2. 在 Prompt 中明确区分：“支持空Tensor”（shape 约束）≠ “参数可选”（指针可为 null）。
  3. 增加基于错误码表的交叉验证：如果参数出现在 ACLNN_ERR_PARAM_NULLPTR 的描述中，则 is_optional 应为 false。
  4. 在 OPTIONAL_EXTRACT_PROMPT 中要求 LLM 返回 src_text（判断依据的原文），便于溯源。
