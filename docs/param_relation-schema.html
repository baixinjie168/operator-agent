概述与风险

当前提取路径

extract_relations.py：ws 和 exe 各 1 次 LLM 调用，从 params + constraints 拼接内容中一次性提取所有参数耦合关系 → merge_relations_node → save_relations_node → build_param_relations_node

核心风险

风险说明
最大 一次性提取遗漏LLM 倾向提取"明显的"表格关系，忽略散文中的隐性关系；输出超过 5-8 项后质量下降；跨 section 信息距离远时关系丢失
中 归因错误长表格中 LLM 可能将第 N 行关系错误归因到相邻行
中 跨 section 丢失params section + constraints section 拼接后，相关信息距离远，LLM 无法关联

为什么用 Agent 循环？ 全局提取天然适合分治策略。一次性提取的召回上限只能通过迭代提取突破，预期 +25-35% 召回率，成本每文档仅 +3-8 次 LLM 调用。

Agent 循环架构

输入：section_content, param_names（从 DB 查询） | v +-- [轮次 1] 分块提取 --------------------------------+ | 将 section 拆分为逻辑块 | | （Markdown 表格 + HTML <table>） | | 对每块 LLM(chunk) → relations[] → 合并去重 | +----------------------------------------------------+ | v +-- [轮次 2] 双重覆盖率检查 -------------------------+ | uncovered_params = 未出现在任何关系中的参数 | | uncovered_paragraphs = 提及 2+ 参数但未引用的段落 | | 两者均为空 → 跳到轮次 4 | +----------------------------------------------------+ | （存在未覆盖） v +-- [轮次 3] 针对性提取 ------------------------------+ | 3a. 未覆盖参数: extract_param_context → LLM | | 3b. 未覆盖段落: paragraph → LLM | | 合并 + 去重 | +----------------------------------------------------+ | v +-- [轮次 4] 自反思（最多 2 轮） --------------------+ | LLM(摘要+描述+引用, section) → 遗漏检查 | +----------------------------------------------------+ | v 输出：all_relations[] + coverage_report

四轮迭代实现

轮次 1：分块提取

v2 支持三种文档格式的分块：

格式特征分块策略占比
HTML 表格<table>...</table>整块，超 15 个 <tr> 则按 <tr> 分组（保留表头）~85%
Markdown 表格连续 | 开头行整块，超 15 数据行则按 10 行分组（保留表头）~6%
参数列表- param（type，dir）：descv2 新增每个参数条目（含子项）为一个块~9%

参数列表格式示例（65 篇文档如 aclnnMoeTokenPermuteWithRoutingMapGrad、aclnnExpand 等）：

  - permutedTokenOutputGrad（aclTensor *，计算输入）：正向输出permutedTokens的梯度，
    非droppad模式要求shape为一个2D的（tokens_num * topK_num，hidden_size）...
    数据类型同permutedProbsOutputGradOptional...
    - <term>Atlas A2</term>：数据类型支持 FLOAT16、FLOAT...
  - sortedIndices（aclTensor *，计算输入）：索引取值范围[0，tokens_num * topK_num - 1]...

# 参数列表项检测正则
_PARAM_LIST_RE = re.compile(
    r"^\s*-\s+(\w+)\s*[（(][^）)]*[）)]\s*[:：]", re.UNICODE)

def _is_param_list_item(line: str) -> bool:
    """检测是否为参数列表项：- param_name（type，direction）：..."""
    return bool(_PARAM_LIST_RE.match(line))

def _is_nested_item(line: str) -> bool:
    """检测是否为参数列表项的子项（更深层缩进的 - 开头行）"""
    stripped = line.strip()
    indent = len(line) - len(line.lstrip())
    return indent >= 4 and stripped.startswith("-")


def split_into_chunks(section_text: str) -> list[str]:
    """拆分策略（v2.1）：
    - HTML 表格：<table>...</table> → 超 15 个 <tr> 则按 <tr> 分组（保留表头）
    - Markdown 表格：连续 | 行 → 超 15 数据行则按 10 行分组（保留表头）
    - 参数列表：每个 - param_name（type，dir）：... 条目（含子项）为一个块
    - 散文：按空行分隔
    - 合并相邻小块（< 1500 字符）
    """
    blocks, current, mode = [], [], "prose"
    lines = section_text.split("\n")
    i = 0
    while i < len(lines):
        line, stripped = lines[i], lines[i].strip()

        # HTML 表格：收集整个 <table>...</table>
        if mode != "html_table" and re.search(r"<table", stripped, re.I):
            if current: blocks.append(current); current = []
            table_lines = [line]; i += 1
            while i < len(lines) and not re.search(r"</table", lines[i], re.I):
                table_lines.append(lines[i]); i += 1
            if i < len(lines): table_lines.append(lines[i])
            tr_idx = [j for j,l in enumerate(table_lines)
                      if re.search(r"<tr[\s>]", l, re.I)]
            if len(tr_idx) > 15:
                hdr_end = _find_tr_end(table_lines, tr_idx[0])
                for s in range(0, len(tr_idx), 10):
                    e = tr_idx[s+10] if s+10 < len(tr_idx) else len(table_lines)
                    blocks.append(table_lines[:hdr_end] + table_lines[tr_idx[s]:e])
            else:
                blocks.append(table_lines)
            mode = "prose"; i += 1; continue

        # Markdown 表格
        if mode == "prose" and stripped.startswith("|"):
            if current: blocks.append(current); current = []
            mode = "md_table"
        if mode == "md_table" and stripped.startswith("|"):
            current.append(line); i += 1
            if len(current) > 17 and "---" not in stripped:
                hdr = current[:2]
                for s in range(0, len(current)-2, 10):
                    blocks.append(hdr + current[2:][s:s+10])
                current, mode = [], "prose"
            continue
        if mode == "md_table":
            if current: blocks.append(current)
            current, mode = [], "prose"

        # 参数列表项：- param_name（type，direction）：description
        if mode == "prose" and _is_param_list_item(line):
            if current: blocks.append(current); current = []
            mode = "param_list"
        if mode == "param_list":
            # 遇到新的顶层参数列表项 → 切分
            if _is_param_list_item(line):
                if current: blocks.append(current)
                current = [line]
                i += 1; continue
            # 子项（嵌套的 - 开头）或续行 → 归属当前参数
            if _is_nested_item(line) or stripped == "" or line[0:1] in (" ", "\t"):
                current.append(line); i += 1; continue
            # 非缩进行（如 "- **返回值：**"）→ 退出参数列表模式
            if current: blocks.append(current); current = []
            mode = "prose"

        # 散文
        if mode == "prose":
            current.append(line)
            if stripped == "": blocks.append(current); current = []
        i += 1
    if current: blocks.append(current)
    return _merge_small_blocks(blocks, max_size=1500)


async def extract_relations_chunked(section_content: str, llm) -> list[dict]:
    chunks = split_into_chunks(section_content)
    all_relations = []
    for i, chunk in enumerate(chunks):
        if not chunk.strip(): continue
        try:
            rels = await _extract_relations(chunk, llm)
        except Exception:
            logger.warning("分块 %d/%d 提取失败", i+1, len(chunks)); continue
        for r in rels: r["_source_chunk"] = i
        all_relations.extend(rels)
    return _dedup_relations(all_relations)  # 复用 merge_relations._dedup_relations

轮次 2：双重覆盖率检查

v2 v1 中 find_uncovered_context_mentions 已定义但从未调用。v2 集成两个检查维度：

  • 参数覆盖率：find_uncovered_params — 找出未出现在任何关系中的参数
  • 段落覆盖率：find_uncovered_context_mentions — 找出提及 2+ 参数但未被 source_citation 覆盖的段落（使用 60 字符指纹模糊匹配 + word-boundary 参数匹配）
展开代码 
def find_uncovered_params(param_names: list[str], relations: list[dict]) -> list[str]:
    covered = {p for r in relations for p in r.get("params", [])}
    return [n for n in param_names if n not in covered]

def find_uncovered_context_mentions(
    section_content: str, param_names: list[str], relations: list[dict]
) -> list[str]:
    # 收集已有 source_citation 的 60 字符指纹
    fragments = set()
    for r in relations:
        c = r.get("source_citation", "").strip()
        for s in range(0, max(0, len(c)-30), 30):
            fragments.add(c[s:s+60])

    uncovered = []
    for para in section_content.split("\n\n"):
        para = para.strip()
        if len(para) < 20: continue
        mentioned = [n for n in param_names
                     if re.search(r"(?<![a-zA-Z0-9_])" + re.escape(n)
                                  + r"(?![a-zA-Z0-9_])", para)]
        if len(mentioned) >= 2 and not any(f in para for f in fragments):
            uncovered.append(para)
    return uncovered

主 Agent 循环

核心轻量级 Agent 循环 — 纯 Python + asyncio

MAX_SELF_CHECK_ROUNDS = 2
MAX_WALL_TIME_PER_DOC = 120  # 秒

async def extract_relations_agent(
    section_content: str, param_names: list[str], llm: ChatOpenAI,
) -> tuple[list[dict], dict]:
    """v2 改进：
    - 顶层 try-except：任何轮次失败都返回已有结果
    - LLM 实例由调用方传入，避免重复创建
    - 复用 merge_relations._dedup_relations
    - 轮次 2 集成段落覆盖率 + 轮次 3 同时处理参数和段落
    """
    from agent.nodes.param_relation_extract.merge_relations import _dedup_relations

    all_relations, report = [], {"round1":0, "round3":0, "round4":0,
                                  "self_check_rounds":0, "total_rounds":0}
    try:
        # 轮次 1：分块提取
        all_relations = _dedup_relations(await extract_relations_chunked(section_content, llm))
        report["round1"] = len(all_relations)

        # 轮次 2：双重覆盖率
        unc_params = find_uncovered_params(param_names, all_relations)
        unc_paras  = find_uncovered_context_mentions(section_content, param_names, all_relations)
        has_targeted = bool(unc_params or unc_paras)

        # 轮次 3：针对性提取（参数 + 段落并发，Semaphore(5)）
        if has_targeted:
            tasks = [_extract_for_param(n) for n in unc_params] \
                  + [_extract_for_paragraph(p) for p in unc_paras]
            for rels in await asyncio.gather(*tasks):
                all_relations.extend(rels)
            all_relations = _dedup_relations(all_relations)
            report["round3"] = len(all_relations) - report["round1"]

        # 轮次 4：自反思（最多 2 轮，单轮失败则 break）
        for rnd in range(MAX_SELF_CHECK_ROUNDS):
            additional = await self_check_relations(llm, all_relations, section_content)
            if not additional: break
            before = len(all_relations)
            all_relations = _dedup_relations(all_relations + additional)
            report["round4"] += len(all_relations) - before
            report["self_check_rounds"] += 1
            if len(all_relations) == before: break

    except Exception:
        logger.exception("Agent 循环异常，回退到已提取 %d 条", len(all_relations))

    # 统计
    report["total_rounds"] = 1 + (1 if has_targeted else 0) + report["self_check_rounds"]
    final_unc = find_uncovered_params(param_names, all_relations)
    report.update(total=len(all_relations), uncovered_params=final_unc,
                  coverage=f"{len(param_names)-len(final_unc)}/{len(param_names)}")
    return _cleanup(all_relations), report

方案原则

  1. 分治提取：拆分为块逐块提取，不一次性提取所有结果
  2. 有目标地迭代：每轮有明确策略（分块 → 针对性 → 自反思）
  3. 成本可控：自反思最多 2 轮，覆盖率驱动提前终止，120s 超时兜底
  4. schema 一致：所有轮次输出统一 schema（含 param_optional），下游零改动
  5. 渐进上线：5 个阶段，每个可独立部署和度量
