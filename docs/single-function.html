一、背景：什么是单函数算子

常规 CANN 算子文档遵循两段式接口模式，包含两个函数：

  • aclnnXxxGetWorkspaceSize(...) —— 第一段，入参校验 + workspace 计算
  • aclnnXxx(workspaceAddr, workspaceSize, executor, stream) —— 第二段，执行计算

而 aclnnCalculateMatmulWeightSize / aclnnCalculateMatmulWeightSizeV2 这类文档只有一个函数：

aclnnStatus aclnnCalculateMatmulWeightSize(
    const aclIntArray *tensorShape,
    uint64_t          *weightTensorSize)

它没有 GetWorkspaceSize 函数，也没有 workspaceSize/executor 框架参数。当前 EXTRACT 流水线在 assemble_result 处硬编码 endswith("WorkspaceSize") / endswith("GetWorkspaceSize") 来识别"主函数"（其余节点经核实不按函数名过滤），导致这类单函数文档的参数在最终装配阶段被全部丢弃。

二、根因：Section Splitter 的分类逻辑

单函数文档的结构如下（以 aclnnCalculateMatmulWeightSize.md 为例）：

# aclnnCalculateMatmulWeightSize          ← H1 标题
## 产品支持情况                            ← H2
## 功能说明                                ← H2
## 函数原型                                ← H2
## aclnnCalculateMatmulWeightSize          ← H2（标题 == 算子名）
  - **参数说明：**                          ← H3（bold list item）
  - **返回值：**                            ← H3
## 约束说明                                ← H2
## 调用示例                                ← H2

section_splitter.py 中 _classify_h2 的关键判断：

def _classify_h2(heading, operator_name):
    if heading_stripped.endswith("GetWorkspaceSize"):
        return SectionType.GET_WORKSPACE_SIZE
    if heading_stripped == operator_name:
        return SectionType.EXECUTE_API      # 单函数文档命中这里

由于 H2 标题等于 operator_name，该 section 被分类为 EXECUTE_API，而不是 GET_WORKSPACE_SIZE。随后 _classify_h3 根据父级类型切换子 section 类型：

if section_type == SectionType.PARAMS_GET_WORKSPACE:
    return (SectionType.PARAMS_EXECUTE
            if parent_type == SectionType.EXECUTE_API   # 父级是 EXECUTE_API 则变成 EXECUTE
            else SectionType.PARAMS_GET_WORKSPACE)
结论：单函数文档最终产生的 section 是：
实际产生的 section两段式文档对应的是否可用
params_execute（含参数表）params_get_workspace有内容
return_codes_executereturn_codes_get_workspace有内容
无 params_get_workspace—不存在
无 return_codes_get_workspace—不存在

即：参数表内容在 params_execute 中，但流水线多处只取 params_get_workspace。

五、适配计划

核心思想：引入"主函数 (primary function)"抽象，封装 GetWorkspaceSize 与单函数的差异。主函数 = 其参数属于真实算子参数的函数：两段式算子为 *GetWorkspaceSize 函数；单函数算子为该唯一函数本身。

Step 1 · 在 param_validators.py 添加共享 helper

文件：packages/agent/src/agent/utils/param_validators.py

集中"单函数模式"判定逻辑，避免 endswith("WorkspaceSize") 散落各处：

def is_single_function_mode(signatures: list[dict]) -> bool:
    """True 当文档不含任何 GetWorkspaceSize 函数（单函数算子）。

    signatures 为空（查询失败/文档解析失败）时返回 False —— 此时无法判定
    模式，调用方应回退到两段式旧行为，避免误判为单函数导致收集非法参数。
    """
    if not signatures:
        return False
    return not any(is_ws_function(s.get("function_name", "")) for s in signatures)


def get_primary_function_names(signatures: list[dict]) -> set[str] | None:
    """返回应视为"主函数"的函数名集合（其参数为真实算子参数）。

    - 两段式算子：返回 GetWorkspaceSize 函数名集合
    - 单函数算子：返回所有函数名（通常只有一个）
    - signatures 为空：返回 None（哨兵值），调用方应回退到
      endswith("WorkspaceSize") 旧行为，避免空集导致参数全丢。
    """
    if not signatures:
        return None
    ws_fns = {s["function_name"] for s in signatures
              if is_ws_function(s.get("function_name", ""))}
    if ws_fns:
        return ws_fns
    return {s["function_name"] for s in signatures if s.get("function_name")}
