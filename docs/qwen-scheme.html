1. 背景与可行性分析

Qwen API 关键信息
项目值
Base URLhttps://dashscope.aliyuncs.com/compatible-mode/v1
协议OpenAI Compatible（/v1/chat/completions）
认证方式Authorization: Bearer <api_key>
模型示例qwen3-max、qwen-plus、qwen-turbo

为什么可以直接复用 ChatOpenAI？

项目中所有节点均通过 langchain_openai.ChatOpenAI 创建 LLM 客户端。该类支持任意 OpenAI 兼容端点——只需传入正确的 base_url 和 api_key。

Qwen 的 DashScope compatible-mode 实现了标准的 /v1/chat/completions 协议，与 ChatOpenAI 完全兼容，无需引入新的 SDK 或 HTTP 客户端。

现有 Provider 架构

LLMProvider (enum)          Settings (config.py)
+----------------+          +----------------------------+
| ZAI            |---+      | zai_api_key / base_url     |--> active_api_key
| DEEPSEEK       |   match  | deepseek_api_key / ...     |    active_base_url
| OWN_AI         |--------->| own_ai_api_key / ...       |    active_model
+----------------+          +----------------------------+         |
                                                                 v
                                                   ChatOpenAI(api_key, base_url, model)

新增 QWEN 只需在左侧枚举和中间配置层各加一条分支，右侧所有节点零改动。

6. Step 4 — 切换使用 Qwen

修改 .env 中的一行即可切换：

LLM_PROVIDER=qwen
QWEN_API_KEY=<your-real-dashscope-api-key>

可选覆盖默认值（取消注释即可）：

QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_MODEL=qwen3-max
切换后无需修改任何代码 所有节点中的 ChatOpenAI(api_key=settings.active_api_key, base_url=settings.active_base_url, model=settings.active_model, ...) 会自动使用 Qwen 的端点、密钥和模型名。

9. 风险与注意事项

模型兼容性 ChatOpenAI 依赖 OpenAI API 协议。qwen3-max 等模型通过 DashScope 的 compatible-mode 完整支持该协议，但部分高级特性（如 function calling、streaming）可能存在差异。建议先用简单 prompt 测试基本调用，再逐步验证 structured output 场景。
温度参数 项目中各节点的 temperature 统一设为 0.1（确定性提取场景）。Qwen 模型对温度参数的行为与 OpenAI 一致，无需特别处理。
速率限制 DashScope 对免费额度和 QPM（每分钟请求数）有限制。如果批量处理大量算子文档，建议关注 DashScope 控制台的用量监控。
备选方案：通过 OWN_AI Provider 快速接入（零代码改动） 如果不想新增枚举值，也可以直接复用现有的 OWN_AI Provider，只需改 .env： 
LLM_PROVIDER=own-ai
OWN_AI_API_KEY=<your-real-dashscope-api-key>
OWN_AI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
OWN_AI_MODEL=qwen3-max
这种方式无需改代码，立即生效。缺点是与 DeepSeek 并列时语义不够清晰，推荐正式使用走新增 QWEN Provider 的方案。
