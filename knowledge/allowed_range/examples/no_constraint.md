# 模式：无取值约束

## 适用场景

参数没有明确的取值范围约束，或者取值范围由其他参数的约束间接决定。

## 识别特征

- 描述中未出现任何取值范围、枚举值、数值限制
- 参数类型为 aclTensor / aclTensorList（维度不等于取值范围）
- 描述只涉及 shape/dtype/format，不涉及值域

## 处理方式

不在输出数组中产出该参数。
（调用方对未出现的参数默认填空数组 []）

## 示例

### 示例 1：Tensor 参数

输入: "计算输入，公式中的输入x。支持输入的维度最少是2维，最多是8维。"
说明: 维度约束 (2-8维) 不是取值范围，是 shape 约束。不提取。
输出: 该参数不出现在结果数组中。

### 示例 2：无值域描述

输入: "表示高精度或者高性能选择。"
说明: 描述未提及具体取值范围。
输出: 该参数不出现在结果数组中。
（注：如果该参数是 int 类型且在文档别处有取值描述，应在别处提取。）

### 示例 3：bool 类型

输入: "transposeX1，是否转置。"
说明: bool 类型，取值固定为 true/false。
输出: 该参数不出现在结果数组中。
（注：bool 类型由代码短路处理，不需要 LLM 提取。）

### 示例 4：字符串长度约束（不应提取为 allowed_range_value）

输入: "字符串长度要求(0, 128)。"
说明: 这是字符串**长度**约束，不是参数取值范围。参数本身是字符串标识符（如通信域名），
其取值无法枚举或限定范围。长度约束应由 constraints_in_parameters 表达（如 `len(group) > 0 and len(group) < 128`）。
输出: 该参数不出现在结果数组中。

输入: "长度范围1-255"
说明: 同样是字符串长度约束，不是取值范围。
输出: 该参数不出现在结果数组中。

**关键区分**：
- "取值范围0-100" → 数值参数的值域 → 提取为 allowed_range_value
- "字符串长度要求(0, 128)" → 字符串参数的长度限制 → 不提取，属于 constraints
- "支持fastgelu/gelu/relu" → 枚举值 → 提取为 allowed_range_value (enum)
